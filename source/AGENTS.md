# Project Instructions

- Treat the current workspace directory as the repository root.
- Keep these instructions portable. Do not store machine-specific absolute paths here.
- Main folders:
  - `backend`: application backend.
  - `database`: Oracle database scripts, packages, migrations, schema objects, and seed data.
  - `frontend`: application frontend.
- For any database-related task, read `database/AGENTS.md` first and follow it.
- Use the Git executable available in the current environment. If `git` is not on PATH, ask for or discover the local Git executable for that machine, but do not persist that local path in repo instructions.
- Use `rg` for searches when available.
- Do not revert unrelated local changes. This repo often has work-in-progress files during migration work.
